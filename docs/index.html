nana

Go-native multi-agent orchestration for OpenAI Codex CLI.

set -euo pipefail

npm install -g @openai/codex

case "$(uname -s)-$(uname -m)" in
  Linux-x86_64) NANA_TARGET="x86_64-unknown-linux-gnu" ;;
  Linux-aarch64 | Linux-arm64) NANA_TARGET="aarch64-unknown-linux-gnu" ;;
  Darwin-x86_64) NANA_TARGET="x86_64-apple-darwin" ;;
  Darwin-arm64) NANA_TARGET="aarch64-apple-darwin" ;;
  *)
    echo "Download the matching native asset from https://github.com/Yeachan-Heo/nana/releases/latest" >&2
    exit 1
    ;;
esac

curl -fsSL -o nana.tar.gz "https://github.com/Yeachan-Heo/nana/releases/latest/download/nana-${NANA_TARGET}.tar.gz"
tar -xzf nana.tar.gz
sudo mkdir -p /usr/local/bin
sudo install -m 0755 nana nana-runtime nana-explore-harness nana-sparkshell /usr/local/bin/
rm -f nana nana-runtime nana-explore-harness nana-sparkshell nana.tar.gz

What's New in 0.11.x

The current docs target the Go-native 0.11.x command surface: native binaries, nana explore, nana sparkshell, and the $deep-interview → $ralplan workflow.

Go-only installation

Install NANA from native release binaries and use the current Go-owned CLI/runtime surface for setup, review, and work automation.

Current launch defaults

Use nana with the configured defaults, add nana --high for a one-off reasoning override, reserve nana --madmax for trusted environments only because it bypasses approvals and sandboxing, or use nana --fast when you want NANA to issue Codex /fast before the rest of your prompt.

Native explore + sparkshell

nana explore is the read-only repo lookup surface, while nana sparkshell gives compact summaries for noisy commands and explicit --tmux-pane inspection.

Workflow-first execution

Clarify with $deep-interview, approve the path with $ralplan, then continue with direct implementation, nana review, or nana work.

Quick Start

  1. npm install -g @openai/codex
  2. Set NANA_TARGET using the platform detection snippet above, or choose the matching release asset manually.
  3. curl -fsSL -o nana.tar.gz "https://github.com/Yeachan-Heo/nana/releases/latest/download/nana-${NANA_TARGET}.tar.gz"
  4. tar -xzf nana.tar.gz
  5. sudo mkdir -p /usr/local/bin
  6. sudo install -m 0755 nana nana-runtime nana-explore-harness nana-sparkshell /usr/local/bin/
  7. rm -f nana nana-runtime nana-explore-harness nana-sparkshell nana.tar.gz
  8. nana setup
  9. nana doctor
  10. nana, nana --high, nana --fast, or nana --madmax in trusted environments only because it bypasses approvals and sandboxing

Architecture

User
  -> Codex CLI
     -> AGENTS.md (orchestration brain)
     -> ~/.codex/prompts/*.md (installable active/internal agent prompt catalog)
     -> ~/.codex/skills/*/SKILL.md (skill catalog)
     -> ~/.codex/config.toml (features, notify, MCP)
     -> .nana/ (runtime state, memory, plans, logs)
